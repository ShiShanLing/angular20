# 移动端架构师岗位要求说明

本文档汇总业界常见的「移动端架构师」（Mobile Architect / Staff Mobile / Mobile Tech Lead）能力模型与职责边界，供招聘、自评或团队培养参考。**具体岗位以各公司 JD 为准**。

---

## 1. 角色定位

移动端架构师承担 **技术选型、全局方案、工程规范与演进**，并与产品、后端、设计协同落地；多数情况下仍需具备 **深度排障与关键代码把控** 能力，而非仅做「纯管理」。

---

## 2. 技术深度

### 2.1 平台能力（至少一端精通）

| 方向 | 典型技能 |
|------|----------|
| **Android** | Kotlin/Java、Gradle、Jetpack、Compose、生命周期与进程模型、厂商适配 |
| **iOS** | Swift、UIKit/SwiftUI、Xcode、签名与证书、App Store 审核与合规要点 |

多数岗位要求 **单端极强 + 另一平台或跨端能做出架构决策**；双原生精通为加分项。

### 2.2 跨端技术（若团队采用）

- Flutter / React Native / Kotlin Multiplatform 等：**渲染与线程模型、原生桥接、性能瓶颈、发布与调试链路**。
- 能基于团队规模与业务复杂度做 **选型、迁移节奏与技术债评估**，而非仅会使用框架 API。

### 2.3 架构与设计

- **模块化**：业务拆分、接口边界、依赖倒置、循环依赖治理。
- **分层与范式**：Clean Architecture、MVVM、MVI、VIPER 等 **适用场景** 与取舍。
- **横切能力**：依赖注入、路由/导航统一、多环境配置、网络层与安全（TLS、证书绑定、本地敏感数据存储）。

### 2.4 性能与稳定性

- **指标**：冷/热启动、首屏、内存、卡顿（帧率）、电量、包体积。
- **平台问题**：Crash / ANR / OOM / Watchdog 等治理思路。
- **工具**：Android Profiler、Systrace、Instruments、线上监控（如 Firebase Crashlytics、自建 APM）及 **问题归因路径**。

### 2.5 工程化与交付

- CI/CD、多渠道/多环境构建、灰度与回滚策略。
- Feature Flag / 远程配置与版本兼容策略。
- 静态检查、代码规范、Code Review 标准。
- 自动化测试策略：**单测 / UI 测试 / 截图测试** 的投入产出与分层。

### 2.6 安全与合规（随业务加重）

- 权限与隐私（采集最小化、清单与披露）。
- 数据传输与存储加密、防篡改与逆向的基本认知。
- 跨境与监管相关要求 **由法务/合规牵头**，架构师需能理解技术落地约束。

---

## 3. 「架构」专属职责

| 领域 | 说明 |
|------|------|
| **技术演进** | 新技术引入节奏、大版本升级方案、遗留系统迁移 |
| **规范与资产** | 编码规范、组件库/设计系统对接、API 契约（如 OpenAPI）、破坏性变更沟通 |
| **复杂领域** | 离线优先、多端同步、音视频、支付、地图/LBS 等的边界与容错 |
| **可观测性** | 埋点体系、日志与指标、线上问题闭环与复盘模板 |

---

## 4. 软技能与协作

- 与 **后端** 对齐版本策略、幂等、超时与降级；与 **产品** 对齐性能预算与交付节奏。
- **方案文档化**：RFC、ADR、里程碑与风险清单。
- **带人与评审**：辅导中级工程师，把控 PR 质量与架构一致性。

---

## 5. 经验与职级（市场常见参考）

- **架构师类 title**：常见期望 **约 8 年及以上** 移动端相关经验（大厂 Staff 等可能更高），并有 **从 0 到 1 或大规模重构/模块化** 的落地案例。
- **中小团队**：资深移动端承担全局架构职责的情况较多，title 未必为「架构师」，但能力模型重叠度高。

---

## 6. 面试常见考察点

1. 模块化如何拆分、多团队并行如何避免耦合失控  
2. 启动优化：冷启动链路、可控项与度量  
3. 列表与复杂 UI 的性能与内存  
4. 网络层设计：拦截器、重试、缓存、一致性与安全  
5. 发布体系：灰度、热修复（若使用）、回滚与监控  
6. 典型线上事故：**归因、止损、复盘与预防**

---

## 7. 可选附录：按团队形态取舍

| 团队形态 | 建议侧重点 |
|----------|------------|
| 双原生 + 大团队 | 模块化、接口契约、构建与 CI、两端对齐规范 |
| 以 Flutter/RN 为主 | 引擎层理解、混合栈、原生插件边界、包体积与渲染性能 |
| 强合规金融等 | 安全加固、渗透配合、日志脱敏、合规审计可追溯 |
